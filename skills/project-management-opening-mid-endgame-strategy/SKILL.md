# Opening, Midgame, and Endgame Strategy

This skill helps AI advise teams on splitting release planning into early risk work,
mid-term feature completion, and final polish.

---

## When to use this skill

Use this skill when you need to:

- organize large features into clear planning phases,
- keep risky, core work at the start of delivery,
- avoid saving all risk to the end,
- ensure the team has room to test and refine,
- communicate a coherent execution strategy.

---

## Outcome

Produce guidance that:

- defines opening game work as the core, risky end-to-end path,
- defines midgame work as filling in additional functionality and quality,
- defines endgame work as refining polish, usability, and readiness,
- explains why each phase should have distinct learning goals,
- helps teams sequence work to reduce uncertainty early.

---

## Instructions for the AI

1. **Describe the opening game**
   - Recommend focusing on essential end-to-end behavior and major risks.
   - Advise excluding optional or polish work.
   - Emphasize that this phase validates whether the solution is technically feasible.

2. **Describe the midgame**
   - Recommend adding supporting functionality, user flows, and business rules.
   - Advise using this phase to discover performance or integration issues.
   - Emphasize that this phase makes the system more complete and usable.

3. **Describe the endgame**
   - Recommend polishing the product, improving UX, and preparing for release.
   - Advise focusing on refinements that matter to users.
   - Emphasize this is the phase for tightening quality and coherence.

4. **Align phases with learning goals**
   - Recommend treating each phase as a different type of learning.
   - Advise teams to use opening game for risk, midgame for completeness,
     and endgame for polish.
   - Encourage teams to adjust the plan if new risk appears in later phases.

---

## Quality criteria

A strong opening/mid/endgame recommendation should be:

- **risk-aware:** risk is addressed early,
- **incremental:** the plan builds toward release logically,
- **transparent:** the team understands what each phase achieves,
- **flexible:** the plan adapts to new findings,
- **focused:** each phase has a clear purpose.

---

## Example prompts

- "How should we break a release into opening, midgame, and endgame work?"
- "What belongs in the opening game versus the endgame?"
- "How do we keep risk front-loaded in our release plan?"
