# Plan Only What You Need

This skill helps AI encourage teams to avoid overplanning and map only the portion of
their product or feature that supports the current discovery and delivery conversation.

---

## When to use this skill

Use this skill when you need to:

- keep planning lightweight for small changes,
- avoid mapping the entire product for a single feature,
- make sure planning effort is proportional to the scope,
- preserve shared understanding without over-engineering,
- keep the team focused on the relevant story.

---

## Outcome

Produce guidance that:

- explains the value of planning just enough,
- helps teams avoid wasting effort on unnecessary detail,
- recommends focusing maps on the current feature or slice,
- encourages conversation over documentation,
- warns against using overplanning as a reason to delay work.

---

## Instructions for the AI

1. **Clarify scope**
   - Ask whether the plan is for a full product or a feature change.
   - Recommend planning only the part needed to support the current discussion.
   - Encourage the team to avoid whole-product mapping when a feature is small.

2. **Keep the map focused**
   - Suggest limiting the map to the relevant user journey and associated dependencies.
   - Recommend including enough context to make decisions, but no more.
   - Advise teams to avoid a false sense of completeness from overlarge maps.

3. **Use the map as a conversation tool**
   - Emphasize that the map’s purpose is shared understanding.
   - Recommend using it to tell the feature story, not as a final specification.
   - Advise updating the map only as new learning justifies it.

4. **Avoid planning paralysis**
   - Warn against delaying delivery because the plan is not comprehensive.
   - Encourage moving forward with a focused plan and revising as needed.
   - Recommend smaller, targeted maps for each distinct feature or slice.

---

## Quality criteria

A strong plan-only-what-you-need recommendation should be:

- **lean:** the plan contains just enough context,
- **relevant:** it supports the current decision,
- **practical:** it avoids unnecessary overhead,
- **conversational:** it emphasizes team discussion,
- **adaptive:** it allows the plan to grow only when needed.

---

## Example prompts

- "How much of our product do we need to map for this small feature?"
- "Explain why we should plan only what we need for the current conversation."
- "How do we avoid overplanning while still preserving shared understanding?"
