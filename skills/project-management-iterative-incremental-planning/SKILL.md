# Iterative and Incremental Planning

This skill helps AI explain the difference between iteration and increment,
and how teams should use both to learn and build a feature safely.

---

## When to use this skill

Use this skill when you need to:

- clarify iterative learning versus incremental delivery,
- plan development so it supports both change and steady progress,
- avoid the trap of pure incremental work without room for learning,
- avoid endless iteration without incremental output,
- help teams balance evaluation and build-up.

---

## Outcome

Produce guidance that:

- explains iteration as repeated evaluation and learning,
- explains increment as adding more capability over time,
- shows how both terms apply in software delivery,
- recommends structures that preserve the calendar while allowing change,
- warns against confusing iteration with unbounded rework.

---

## Instructions for the AI

1. **Define terms clearly**
   - Explain iteration as repeating the same process and improving the product.
   - Explain increment as building up the product in layers toward completion.
   - Emphasize that both are needed in healthy software development.

2. **Use examples**
   - Compare iteration to sketching and refinement.
   - Compare increment to adding layers of functionality.
   - Illustrate with a feature that starts as a simple path and becomes more complete.

3. **Recommend planning habits**
   - Advise teams to plan for learning checkpoints and also for incremental build-up.
   - Recommend preserving enough structure to measure progress without freezing design.
   - Suggest using small slices to improve estimate accuracy and learning velocity.

4. **Warn against anti-patterns**
   - Flag pure incremental work that ignores new information.
   - Flag endless iteration that prevents measurable delivery.
   - Encourage teams to keep both the calendar and the product goals in view.

---

## Quality criteria

A strong iterative/incremental planning recommendation should be:

- **balanced:** both learning and delivery are respected,
- **clear:** terms are defined and not conflated,
- **practical:** planning includes both feedback and progress,
- **adaptive:** the plan changes as real information arrives,
- **predictable:** incremental output keeps the team honest.

---

## Example prompts

- "Explain how to use iterative learning and incremental delivery together."
- "How should a team balance exploring assumptions with building actual functionality?"
- "What are the risks of treating iteration and increment as the same thing?"
